# 민간투자사업 적용 지수 분석

가격기준시점을 100으로 재지수화하여 **소비자물가지수·건설공사비지수·건설투자 디플레이터**의
상대 추이를 비교하는 정적 웹 도구입니다. 민간투자사업에서 소비자물가지수로 건설이자 등을
보정할 때, 실제 공사비 상승과의 격차를 가늠하는 **참고용**으로 사용합니다.

| 지수 | 표 ID | 기관 | 주기 |
|---|---|---|---|
| 소비자물가지수 | `DT_1J22003` | 통계청 (101) | 월 |
| 건설공사비지수 | `DT_39701_A003` | 한국건설기술연구원 (397) | 월 |
| 국내총생산 지출 디플레이터 中 건설투자 | `DT_200Y112` | 한국은행 (301) | 분기 |

## 동작 구조

```
GitHub Actions (월 1회/수동)
   └─ fetch_kosis.py  ──KOSIS OpenAPI(키는 Secrets)──▶  data/indices.json 커밋
GitHub Pages (정적)
   └─ index.html  ──fetch──▶  data/indices.json  ──▶  재지수화·차트·표·분석
```

API 키는 GitHub Secrets에만 두고 페이지에는 노출하지 않습니다. 페이지는 미리 받아둔
JSON만 읽어 브라우저에서 분석하므로 CORS·키 문제가 없습니다.

## 설치

1. 이 폴더를 새 GitHub 저장소에 올립니다(루트에 `index.html`, `data/`, `fetch_kosis.py`, `.github/`).
2. **KOSIS 인증키 발급**: <https://kosis.kr> 회원가입 → 공유서비스 OpenAPI 인증키 신청(회원당 1개).
3. **Secret 등록**: 저장소 → Settings → Secrets and variables → Actions → New repository secret
   - Name: `KOSIS_API_KEY`, Value: 발급받은 키
4. **Pages 활성화**: Settings → Pages → Source를 `Deploy from a branch`, 브랜치 `main` / 폴더 `/ (root)`.
5. **첫 수집 실행**: Actions 탭 → `KOSIS 지수 갱신` → Run workflow.
   완료되면 `data/indices.json`이 실제 수치로 갱신되고, Pages 주소에서 바로 확인됩니다.

수집 전에는 페이지가 **샘플 데이터**(상단 안내 배너 표시)로 동작하므로 화면·로직을 미리 점검할 수 있습니다.

## 사용

- **시점**: 가격기준시점(협약상 기준일이 속한 월). 이 시점이 100이 됩니다.
- **종점**: 비교 종료월. 기본값은 자료 최신월.
- **분석** 버튼 → 세 지수의 재지수 추이 그래프, 시점 대비 종점 변동 표, 분석 문구가 표시됩니다.
- 분기 자료(디플레이터)는 입력한 월이 속한 분기값을 사용해 월 시계열에 정렬합니다.

## 항목 매칭 조정 (선택)

`fetch_kosis.py`는 정확한 코드 대신 한글 항목명·분류명으로 필요한 시계열을 골라냅니다.
표 구조가 바뀌어 매칭에 실패하면 Actions 로그에 **표에 존재하는 라벨 목록**이 출력됩니다.
그 라벨을 보고 `CONFIG`의 `include` 키워드만 수정하면 됩니다. 예)

```python
"cpi": { ..., "include": ["총지수"] }          # '총지수'가 포함된 행 선택
"deflator": { ..., "include": ["건설투자"] }   # '건설투자' 분류 행 선택
```

`EXCLUDE`(등락률·증감·전월·전년 등)에 걸리는 행은 지수 수준값이 아니므로 자동 제외됩니다.

산식(가격기준시점, 적용시점, 가중방식 등)을 우선합니다.
